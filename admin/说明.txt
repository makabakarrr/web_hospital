# 初始数据
项目启动时，自动在数据库创建50条医护人员数据，密码统一为a123456名字是随机生成的（通过查看数据库能看到具体信息，比如 周秀英 是自动创建的医护数据，我通过查看数据库看到用户名是周秀英，然后我就可以用这个账户登录），还会自动创建一个管理员账号{用户名：admin,密码:a123456}
数据库内没有普通用户的账号

# 关于注册
只能注册普通用户账号和医护人员账号，管理员账号只有一个

# 预约
1. 同一个身份证在同一时间段只能预约一次，再次预约会返回预约失败信息。如果第一次预约信息填错，患者先我的预约页面撤销上一次预约，然后返回预约页面重新预约
2. 为方便测试，已设置一个医生在一个时间段只能接受三个预约
3. 页面的表单做了正则验证，需要按正确格式填写。只有验证成功才能发送请求创建预约单，否则会做出提示通知用户进行修改
4. 预约可选择的时间为未来两周，不包括今天
5. 预约的单有四种状态：-1 已过期 0 待处理 1已处理 2已撤销
  患者可以将自己的预约单撤销
  医护人员可以处理 患者选择自己为医生 的预约单， 也有像普通用户创建预约单撤销自己创建的预约单功能
  管理员只能查看所有预约单 不能对预约单进行处理，也不能创建预约单
6. 验证预约单过期：为了方便验证，使用的是电脑的系统时间，所以通过修改电脑的时间，可以验证预约单是否过期。比如我登录的时间是2020-04-12，然后去创建一些不同预约时间的预约单，然后改变电脑的时间，去我的预约页面查看预约单。需要注意的是，只能将电脑的时间修改为登录时间未来一周内的时间（登录时间为12号，修改电脑的时间不能超过19号）。因为存储的登录态一周后会过期，就需要重新登录

# 关于权限
权限分为页面权限和数据权限
页面权限是前端处理的，数据权限是服务器处理的。就是说前端可以通过admin账号给角色分配任何页面，但是不同的角色在每个页面看到的数据是不一样的，而且在每个页面上拥有的功能也不一样
权限管理的功能。这个页面是专为管理员所用的，但是硬要添加权限给其它角色也可以。管理员可以给每个角色都添加这个权限管理页面的权限，每个角色都能看到这个页面。但是只有管理员能够成功设置权限，其它角色不能设置权限，并且会在发送请求后弹出提示框，提示用户找管理员来设置权限
预约医生功能。管理员不能预约医生，如果给了管理员这个页面的权限，管理员是可以看到这个页面，也能填写表单，在提交表单后，不能创建预约单，并会提示管理员不能预约，切换到其它账号重新预约
处理预约单功能（在挂号信息处理这个页面），这个页面是专为医护人员和管理员所用的，但是演示的时候添加给普通用户也可以。每个角色进来看到的数据不一样，医护人员进来看到的是挂在自己名下的预约单，管理员进来看到的是所有的预约单，普通用户进来看到的是他自己创建的预约单，就是跟我的预约页面的数据一样。但是只有医护人员点击处理才有效，其它角色点击不会成功，并且会做出提示
